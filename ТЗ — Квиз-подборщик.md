# ТЗ: Премиальный квиз-подборщик для ROSA

## Контекст

ROSA — премиальный коттеджный посёлок на Новой Риге, средний чек 17–60 млн ₽. Сайт на Tilda, фирстиль: тёмно-синий `#051C2B`, винно-красный `#AB182D`, шрифт Formular (запасной — Inter). Аудитория — обеспеченные семьи 35–55 лет, iPhone, привыкли к Apple-уровню интерфейсов.

## Задача

Сделать встраиваемый в страницу квиз, который:
1. Помогает посетителю найти «свою» коллекцию домов (Residents / Берёзовая / Луговая).
2. Захватывает контакт на финальном шаге.
3. Выглядит как часть премиум-бренда, а не как поп-ап «пройди опрос и получи скидку».

## Что НЕ делать

- Popup или модалка — квиз встраивается в страницу (inline-секция).
- Marquiz-подобная стилистика, эмодзи, гиф-анимации, восклицательные знаки.
- «Получите скидку 5% за прохождение» — никаких приманок, это премиум.
- Прогресс-бар жирной цветной полосой. Шкала есть, но скромная.
- Дефолтные браузерные радио-кнопки/чекбоксы.
- Тяжёлые библиотеки и фреймворки.

## Технические требования

- **Один HTML-файл** с inline CSS и vanilla JS. Не React, не Vue.
- Никаких внешних зависимостей. Шрифт можно подключить с Google Fonts (Inter) — единственное исключение.
- Совместимо с блоком «HTML-код (T123)» в Tilda — просто скопировать содержимое в блок.
- Размер итогового файла — до 30 КБ без шрифта.
- Mobile-first: 75% трафика — мобайл.
- Поддержка iOS Safari, Chrome, Firefox, последние 2 версии.
- Никаких `localStorage` / `sessionStorage` — может конфликтовать с Tilda.

## UX-логика

### Шаги (5 экранов вопросов + 1 экран контактов + 1 экран результата)

**Шаг 1. Для кого вы выбираете дом?**
- Для семьи с детьми
- Для пары
- Для родителей
- Как инвестиция

**Шаг 2. Сколько человек будет постоянно проживать?**
- 1–2
- 3–4
- 5 и больше

**Шаг 3. Какой бюджет рассматриваете?**
- До 25 млн ₽
- 25–40 млн ₽
- 40+ млн ₽
- Нужен расчёт ипотеки

**Шаг 4. Когда планируете заселение?**
- Готов сейчас — нужен готовый дом
- Через 6–12 месяцев
- Через 1–2 года

**Шаг 5. Контакт**
- Имя (обязательно)
- Телефон (обязательно, валидация +7…)
- WhatsApp / Telegram (опционально, поле или toggle)

**Финальный экран. Ваш выбор**
- Название рекомендованной коллекции крупно
- 1–2 фото домов (placeholder-картинки на этапе прототипа)
- 3–4 пункта характеристик (метраж, участок, цена от, срок строительства)
- Сводка ответов пользователя в одну строку
- CTA-кнопка «Обсудить с менеджером» (на этапе прототипа — `console.log(answers)`)

### Алгоритм подбора

```
если бюджет = "40+ млн" ИЛИ "5+ человек" → Residents
иначе если бюджет = "25–40 млн" ИЛИ "3–4 человека" → Берёзовая
иначе → Луговая
```

Если пользователь выбрал «Нужен расчёт ипотеки» — в результате подсветить ссылку на калькулятор.

## Дизайн-гайд

### Цвета
- Фон: `#FFFFFF`
- Текст основной: `#1A1A1A`
- Текст приглушённый: `#6B7280`
- Акцент (выбранное / активное): `#051C2B` (фирменный navy)
- Ошибка / редкий акцент: `#AB182D` (фирменный wine)
- Границы / разделители: `#E5E7EB`

### Типографика
- Шрифт: Inter (или Formular если доступен). 400 / 500 / 600.
- Заголовок шага (вопрос): 28–32px desktop, 22–24px mobile, weight 600
- Подпись варианта: 16–18px, weight 500
- Body: 15–16px, weight 400
- Все размеры в `clamp()` для плавной адаптивности

### Раскладка

**Desktop**
- Контейнер: max-width 720px, центрирован
- Padding: 48px вертикальный, 24px горизонтальный
- Варианты ответов: 2×2 сетка для шагов 1, 3 (4 опции); 1 ряд из 3 для шагов 2, 4 (3 опции)
- Карточка варианта: высота min 96px, padding 20px

**Mobile (≤640px)**
- Варианты: одна колонка, высота карточки min 72px
- Уменьшенные отступы

### Анимация
- Переход между шагами: fade + лёгкий slide (10px), 200мс, easing `ease-out`
- Hover на варианте (desktop): тонкое поднятие тени, граница меняет цвет на navy
- Выбор варианта: short scale 0.98 → 1.0 на 150мс
- Никаких bounce, spring, тряски, конфетти.

### Прогресс
- Тонкая горизонтальная полоса 2px сверху контейнера
- Заполнение navy цветом по мере прохождения
- Опционально: «Шаг 3 из 5» приглушённым текстом справа
- Без процентов, без таймеров

### Кнопки
- Назад: текстовая, не кнопка, размер мелкий, цвет приглушённый
- Далее (только когда выбран ответ): кнопка-прямоугольник, navy фон, белый текст, 48px высота, 24px горизонтальный padding
- На шаге контакта: кнопка «Отправить» того же стиля

## Submit на финальном шаге

На этом этапе — заглушка:

```js
function submitQuiz(answers) {
  console.log('Quiz submitted:', answers);
  // TODO: integration with Tilda form will be added in next step
  showResultScreen(answers);
}
```

Структура `answers`:
```js
{
  audience: 'family' | 'couple' | 'parents' | 'investment',
  size: '1-2' | '3-4' | '5+',
  budget: 'under-25' | '25-40' | '40-plus' | 'need-mortgage',
  timing: 'now' | '6-12-months' | '1-2-years',
  name: 'string',
  phone: 'string',
  messenger: 'wa' | 'tg' | null,
  recommended: 'residents' | 'birch' | 'meadow'
}
```

## Что должно получиться

Один HTML-файл, который:
- Открывается в браузере без сборки — рабочая демка
- Копируется в Tilda T123-блок и сразу работает
- Выглядит как часть сайта ROSA, а не «прикрученный квиз»
- Проходится за 30–40 секунд
- Не вызывает мысль «это явно AI-сгенерированная штука»
